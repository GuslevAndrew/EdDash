# EdDash

EdDash — локальний MVP українського вебзастосунку для аналізу відкритих даних про здобувачів освіти у закладах вищої освіти України.

Основна ідея архітектури:

```text
API ЄДЕБО -> скрипт імпорту -> локальна база даних -> внутрішній API EdDash -> дашборд
```

Застосунок не звертається до ЄДЕБО під час кожного відкриття дашборду. Дані читаються з локальної SQLite-бази.

## Призначення

EdDash допомагає вступникам, батькам, освітнім консультантам, працівникам ЗВО, журналістам і дослідникам аналізувати:

- найбільші університети за кількістю здобувачів;
- популярність спеціальностей;
- динаміку за датами зрізів;
- відмінності між регіонами;
- розподіл за освітніми рівнями та основою вступу;
- результати імпорту відкритих даних.

EdDash не є офіційним сервісом ЄДЕБО. Дані мають інформаційно-аналітичний характер.

## Стек

- Next.js App Router
- TypeScript strict mode
- Tailwind CSS
- Prisma ORM
- SQLite для локальної розробки
- Recharts
- Zod
- `fetch` для HTTP-запитів

## Встановлення

```bash
npm install
```

## Налаштування `.env`

Скопіюйте приклад:

```bash
copy .env.example .env
```

Локальне значення:

```env
DATABASE_URL="file:./dev.db"
EDBO_API_BASE_URL="https://registry.edbo.gov.ua"
EDBO_USER_AGENT="EdDash MVP local analytics contact: local-dev"
```

Файл `.env` не додається до Git.

## База даних

Згенерувати Prisma Client:

```bash
npm run db:generate
```

Створити SQLite-базу та міграцію:

```bash
npm run db:migrate
```

## Демонстраційні дані

Якщо API ЄДЕБО недоступний або потрібна швидка перевірка інтерфейсу:

```bash
npm run db:seed
```

Демо-дані позначені тестовими назвами закладів та довідників.

## Перевірка API ЄДЕБО

```bash
npm run edbo:test
```

Скрипт перевіряє:

- API регіонів;
- API спеціальностей;
- endpoint здобувачів;
- кілька дат, освітніх рівнів і основ вступу;
- HTTP-статус, тип вмісту, час відповіді;
- JSON/HTML/text/error;
- перші записи без надмірного виводу.

Необроблені приклади відповідей і `summary.json` зберігаються в:

```text
data/api-tests/
```

## Імпорт реальних даних

Довідник регіонів:

```bash
npm run edbo:regions
```

Довідник спеціальностей:

```bash
npm run edbo:specialities
```

Довідник закладів вищої та фахової передвищої освіти:

```bash
npm run edbo:institutions
```

Безпечний імпорт одного зрізу здобувачів:

```bash
npm run edbo:import -- --dt 01.10.2023 --qf 1 --eb 40
```

Доступні параметри:

- `--dt` — дата зрізу у форматі `DD.MM.YYYY`;
- `--qf` — освітній рівень;
- `--eb` — основа вступу;
- `--sp` — спеціальність;
- `--rg` — регіон;
- `--id` — заклад освіти.

Імпорт створює запис `ImportRun`, не дублює наявні записи та оновлює змінені записи.

## Локальний запуск

```bash
npm run dev
```

Основні сторінки:

- `/` — головна сторінка;
- `/dashboard` — аналітичний дашборд;
- `/specialities` — довідкова сторінка для вступників з галузями знань і спеціальностями;
- `/admin/import` — локальна технічна сторінка імпорту.

## Структура проєкту

```text
app/
  api/                внутрішні API routes
  dashboard/          сторінка дашборду
  admin/import/       сторінка імпорту
components/
  dashboard/          фільтри, картки, графіки, таблиця
  layout/             оболонка застосунку
  ui/                 базові UI-компоненти
lib/
  dashboard/          серверні запити й фільтрація
  demo/               seed демонстраційних даних
  edbo/               клієнт, схеми, нормалізація API ЄДЕБО
  utils/              форматування
prisma/
  schema.prisma       модель даних
scripts/
  *.ts                тест API та імпортери
data/api-tests/       збережені приклади відповідей API
```

## Довідник галузей і спеціальностей

Сторінка `/specialities` показує довідкову таблицю для вступників на основі чинного переліку галузей знань і спеціальностей з постанови КМУ №266.

Оновити локальний довідник з офіційної HTML-таблиці:

```bash
npm run catalog:specialities
```

Згенеровані дані зберігаються в:

```text
lib/specialities/catalog.ts
```

## Фактична структура відповіді API

Проєкт не припускає наперед точну структуру JSON ЄДЕБО. Скрипт `edbo:test` зберігає реальні відповіді в `data/api-tests/summary.json`.

Під час перевірки 8 червня 2026 року endpoint здобувачів з `exp=json` повернув JSON-масив з українськими назвами колонок:

- `Станом на`;
- `Освітній ступінь`;
- `Основа вступу`;
- `Код спеціальності`;
- `Назва спеціальності`;
- `Регіон`;
- `Код`;
- `Назва закладу освіти`;
- `Денна (бюджет)`;
- `Денна (контракт)`;
- `Заочна (бюджет)`;
- `Заочна (контракт)`;
- `Вечірня (бюджет)`;
- `Вечірня (контракт)`.

EdDash рахує `studentsCount` як суму колонок за формами навчання. Довідники регіонів і спеціальностей на момент перевірки повертали XLSX навіть з `exp=json`, тому імпортери довідників мають fallback-парсинг Excel-файлів через `xlsx`.

Якщо API не повертає певну характеристику, її не треба вигадувати; вона має залишатися відсутньою або технічною fallback-міткою.

## Відомі обмеження MVP

- Масовий історичний імпорт не автоматизований навмисно.
- Немає авторизації для `/admin/import`, бо це локальний технічний екран.
- SQLite використовується лише як локальна база.
- Нормалізатор підтримує підтверджені українські поля endpoint здобувачів і типові fallback-ключі.
- У цьому середовищі стандартний `prisma migrate dev` повертав порожню помилку schema engine, тому для локальної перевірки додана fallback-команда `npm run db:push:local`, яка створює SQLite-схему через Prisma Client.
- Немає офіційної гарантії актуальності або повноти даних.

## Майбутнє перенесення

Архітектура підготовлена до переходу:

```text
SQLite -> PostgreSQL або Supabase
локальний запуск -> Vercel або інший хостинг
ручне оновлення -> автоматичне періодичне оновлення
```

Для PostgreSQL потрібно буде змінити `datasource` у Prisma, оновити `DATABASE_URL`, виконати міграції та перенести механізм імпорту в scheduled job або окремий безпечний backend-процес.
